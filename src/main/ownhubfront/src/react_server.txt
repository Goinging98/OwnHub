리액트 주고받는 정보

Board.js
- 글쓰기 칸으로 글 작성하는 폼을 서버로 전달( 제목, 내용 )
- 작성자, 작성시간은 서버단에서 해결할 것

BoardList.js
- 자유게시판, 부서게시판 등 버튼을 클릭하면 해당 게시판이름을 서버로 전송
- 서버는 전송받은 게시판이름에 해당하는 게시물들을 객체로 전달
- 객체
Content.title - 제목
           .content - 내용
           .wdate - 작성일자
           .cteam - 부서명
           .cuid - 해당 유저 식별키

Git_main.js
- 예시 리포지토리 클릭하면 해당 리포지토리이름을 서버로 전송 후 해당 git 객체를 반환해줄것임.

Git_my.js
- 리포지토리 생성 클릭시 리포지토리 이름을 서버로 전송 사용자 정보는 서버단에서 처리

Member.js
- 검색한 이름을 서버로 전송, 그럼 해당하는 이름을 가진 사원 정보를 List 객체로 전달할 것
- Member[0].username
 		.email
		. auth
		.team
		
Mypage.js
- 마이페이지 버튼 클릭시 서버에서 세션 유저식별값을 불러와서 해당 유저 정보 객체를 반환
- 객체 각 속성 이름은 Member.js에 예시와 같음

signup.js
- 회원가입 시 이름, 아이디, 패스워드, 이메일, 소속부서, 직급 을 서버로 전송 나머지는 서버가 처리

Working_hour.js
- 휴가 시작일자, 종료일자, 종류, 사유 의 정보를 서버로 전송 서버에서 저장하고 처리 불가능할 시 fail 을 보내고 성공하면 sucsess를 보낼 것

Working_Monthly.js
- 무슨파일인지는 모르겠으나 달력인것 같아서 적음 
- 달력페이지 진입하면 해당 유저의 정보를 서버에서 세션값을 조회해서 휴가신청 혹은 근무 시간등을 List 객체로 전달할것임. 
- 그럼 리액트에서 해당 객체의 값을 조회해서 일자별로 휴가표시, 근무 표시를 해주면 됨.
- 리스트 객체 접근은 
	Work[0].sdate -시작날짜
		.edate -종료날짜
		.stitle -휴가종류(연차,반차)
		.scontent - 휴가사유 임
